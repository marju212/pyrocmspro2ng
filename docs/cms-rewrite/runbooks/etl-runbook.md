# ETL runbook

Operational procedure for migrating bockavel data from PyroCMS (`bockavel_legacy`) to Twill (`bockavel_new`). Built during Epic 10; rehearsed before Epic 11 cutover.

## Orchestrator

`php artisan migrate:from-pyrocms [--dry-run] [--only=<entity>] [--source=<db>] [--target=<db>]`

Idempotent: rerunning truncates target tables for migrated entities and re-imports. Foreign keys deferred during run; checked at end.

## Order of operations (dependency-driven)

1. `groups` — must come before members (members ref groups).
2. `members` — must come before ads, blog comments, restricted file ownership.
3. `pages` — must come before blocks (block transforms reference pages for `{{ pages:link }}`).
4. `page_chunks` → blocks under each page.
5. `blog_categories`, `blog_tags`.
6. `blog_posts`.
7. `comments`.
8. `files` — physical files copied to disk first, then DB rows.
9. `restricted_files` — subset of files tagged with `required_groups`.
10. `ad_categories`.
11. `ads` (+ `ad_images` to disk).
12. `news`.
13. `forms` + `form_submissions`.
14. `snippets`.
15. `settings` + `variables`.
16. `redirects` (legacy `default_redirects` table).

## Lex transform

During pages / blog / page_chunks / snippets ETL, body content runs through `LexParser` (Epic 9). See `specs/lex-tag-audit.md`.

## File transfers

- Public files from legacy `uploads/<site>/files/` → S3 public bucket / `storage/app/public`.
- Restricted files (group-tagged) → S3 private bucket / `storage/app/private`.
- Image variants regenerated by Twill on first request (no preprocessing).

## Reconciliation

After dry-run, output report:

| Entity | Legacy count | Target count | Delta | Notes |
|---|---|---|---|---|

Manually inspect deltas before authorizing the live cutover ETL.

## Sample verification

For each entity, pick 3 random IDs and `diff` legacy view vs. target view (HTML). Acceptable diffs: design changes, slug normalization. Unacceptable: missing fields, broken links, lost images.

## Rollback

- ETL writes only to `bockavel_new`. Rolling back = `DROP DATABASE bockavel_new`.
- Files copied to S3 are versioned — old versions accessible.
- Cutover is reversible until the DNS swap. After DNS swap, rollback = swap DNS back + accept loss of any new member registrations / form submissions made on the new site (~minutes window).
